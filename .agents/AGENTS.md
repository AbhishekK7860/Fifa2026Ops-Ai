## Reporting Format Rule

Going forward for every remaining phase (5 through 8): completion reports must always follow the Phase 1-3 format:
- File-by-file breakdown of what was built.
- Exact named constants/functions touched.
- Actual command output (tsc --noEmit, lint, test — real pass/fail counts, not "clean compilation" or "fully implemented").
- Specific test case inputs/outputs.
A narrative highlights summary pointing at a separate walkthrough artifact is not an acceptable completion report.
